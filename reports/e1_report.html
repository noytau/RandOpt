RandOpt · SSL Vision · Experiment E1

Image Classification on ImageNet-C: kNN / Linear Probe / Finetune / RandOpt

Four adaptation rungs on corrupted ImageNet, same splits, same metric — how close does training-free RandOpt come to gradient baselines?

DINOv2-giant 1.136B · ImageNet-C severity 3 · splits 25/10/15 per class · test = 15,000 images (±0.4pp SE) · 2026-07-13 → 07-15 · W&B project randopt, runs e1-*

Headline finding so far. Full-model finetuning moved the weights a distance equivalent to σ ≈ 2.3×10⁻⁴ and gained +2.24pp. RandOpt sampled 100 random directions at exactly those radii (σ = 1e-4 and 3e-4) and its best ensemble gained +0.03pp. Better models exist at the searched radii — random isotropic sampling doesn't find them. At N=100 this is a needle, not a thicket, on a task with 24pp of headroom. (Paper-scale N is the remaining test: N=100 is 50× below the paper's 5,000.)

Dataset — ImageNet-C

ImageNet-C (Hendrycks & Dietterich 2019) applies 15 corruption types × 5 severity levels to the ImageNet-1k validation set (50,000 images = 1,000 classes × exactly 50 per class). Each (corruption, severity) pair is therefore a complete 50k copy of the set, shipped as 224×224 JPEGs — DINOv2's native input size. Canonical ImageNet-C usage is evaluation-only (train on clean ImageNet, test on corruptions), so no standard train/val/test split of it exists; our supervised-adaptation setting defines its own. Every experiment adapts to one (corruption, severity) pair at a time — mixing corruptions in one adaptation would ask a different (harder) question and would pool 15 distortions of the same underlying photos.

Loading & splits (data_handlers/imagenet_c.py)

The handler does two jobs. _ensure_downloaded(data_dir, corruption, severity) is the idempotent fetch: if <data_dir>/<corruption>/<severity>/ already has files it returns immediately; otherwise it downloads the right family tar (wget -c, resumable) and extracts only the requested subtree. load_data(path, split) then builds the splits:

# path encodes the cell:  data/imagenet_c/<corruption>/<severity>
bounds = {"train": (0, 25), "val": (25, 35), "test": (35, 50)}   # per class
rng = np.random.default_rng(split_seed=42)                # fixed seed
for label, wnid_dir in enumerate(sorted(class_dirs)):     # 1000 classes,
    order = rng.permutation(files_of(wnid_dir))           #   wnid-sorted labels
    picks = order[lo:hi]                                  # contiguous slice
    items += [{image_tensor, class_id=label} for i in picks]

Properties this construction guarantees: splits are class-stratified (exactly 25/10/15 from each class → 25k/10k/15k), disjoint by construction (contiguous, non-overlapping slices of one per-class shuffle), and deterministic (same seed → bit-identical membership across every job and method, so all four rungs see literally the same train/test images). Labels come from sorted-wnid order, which matches the standard ImageNet class index — internal consistency is all kNN needs, and a pretrained-head variant later gets correct labels for free. Val exists in every load but is read only by sweep-tier runs; POCs never touch it, and test is evaluated once per experiment. 15k test images put the binomial 1-SE at ±0.4pp near 75% accuracy — the noise floor all gains are judged against.

Common setup

Backbone: facebook/dinov2-giant — SSL-pretrained, no classification head; frozen-backbone embeddings are cached once per split (vision/features.py: one forward stores both the CLS token for kNN and the pooler output for the probe), which makes kNN sweeps and probe training near-free. All runs log to W&B project randopt with the e1- prefix; per-run links in the summary table below.

1 · kNN — the base rung (training-free readout)

What it tests: how well the frozen backbone's embeddings classify corrupted images by nearest neighbors. This number is the base every other method is measured against, and the run doubles as the headroom check.

# gallery = train-split embeddings (frozen backbone, cached)
g = normalize(cls_embeddings(train_images))        # (25000, 1536)
q = normalize(cls_embeddings(query_images))        # val or test
sim = q @ g.T
topv, topi = sim.topk(k)                           # k nearest gallery items
w = exp(topv / tau)                                # DINO-style weighted vote
votes[c] = sum(w  where gallery_label == c)
prediction = argmax(votes)

Result: bases 75.0–77.4% across the four corruptions; accuracy is flat for k ≥ 10 (plateau spread ≤ 0.15pp), so the base is robust rather than tuned. All corruptions leave >20pp headroom — every rung has room to show a gain.

2 · Linear probe — Baseline 2 (head-only gradients)

What it tests: the cheapest gradient adaptation — a linear classifier trained on frozen features. Its trained head also seeds the finetune rung.

3 · Finetune — Baseline 1 (backbone gradients)

What it tests: the strongest standard adaptation, at two scopes — full model (1.138B trainable) vs last-4 blocks + head (115M). Logs ‖Δw‖ converted to an equivalent σ (= ‖Δw‖/√d), making the gradient's travel distance directly comparable to RandOpt's sampling radius.

model = Dinov2Model(giant);  head.load(probe_head)   # warm head protects backbone
trainable = last_n_blocks(model, N) + head           # N=0 → full model
bs  = find_max_batch()                # OOM backoff, incl. AdamW state buffers
opt = AdamW(trainable, lr=1e-5);  cosine_schedule(total_steps)
for 2 epochs, bf16 autocast:
    CrossEntropy(head(model(x)), y).backward();  opt.step()
sigma_equiv = ||w − w₀|| / sqrt(d_trainable)

Result (gaussian_noise): full model 77.81% (‖Δw‖ = 7.88, σ-eq 2.3e-4, 42 min) · last-4 blocks 77.78% (‖Δw‖ = 1.43, σ-eq 1.3e-4, 24 min). Fewer layers matched full FT at half the cost — the adaptation signal lives in the top of the network. The finetuned backbone's own kNN readout also improved (75.57 → 76.6), so the features moved, not just the head.

4 · RandOpt — the contender (no gradients)

What it tests: the Neural-Thickets claim transplanted to SSL vision: are better task models densely packed around the pretrained weights, findable by random perturbation + selection? The σ-screen maps the noise-scale landscape and locates the collapse cliff.

for sigma in {1e-4, 3e-4, 1e-3, 3e-3}:               # one GPU job per cell
    for seed in 100 seeds:
        perturb(w += sigma · N(0,1, seed))            # all 1.136B params
        score = knn_top1(gallery_1k, queries_1k)      # re-embedded per pert.
        restore(w)                                    # bit-exact
top_K = best_K_by_train_score(pooled)                 # K ∈ {1, 5}
for m in top_K:
    preds[m] = knn_predict(full_train_gallery, test)  # 25k gallery re-embedded
ensemble = majority_vote(preds);  report accuracy(ensemble)
scope
all — every backbone parameter (1,136,482,305)
sigma
{1e-4, 3e-4, 1e-3, 3e-3}, N=100 per cell, seeds 201–204
selection
rank by kNN top-1 on 1k train queries vs 1k train gallery; top-K, K ∈ {1, 5}
honesty
selection touches only train; test evaluated once by the ensemble; single-perturbation maxima not quotable (split design queued for review with Nimrod)
cost
~78–106 s/perturbation; ~33 min per ensemble member; 5–7h per cell

Result: ensembles gain +0.03 / −0.04 / −0.53 / −9.28pp at σ = 1e-4 / 3e-4 / 1e-3 / 3e-3. Mean scoring shift collapses from +0.14pp to −24.4pp between σ=1e-3 and 3e-3 — the collapse cliff for giant on this task sits in that interval.

What we are searching for in these results

  1. Does a thicket exist around the SSL init on a task with headroom? The clean comparison E1 was built for: gradient descent reaches +2.2pp at a travel distance of σ-eq ≈ 1.3–2.3e-4; random sampling at those very radii (100 draws each) finds +0.03pp at best. Existence ≠ findability: good models are there, but occupy negligible volume. Verdict at N=100: needle. The remaining test before concluding is scale — the paper uses N=5,000; the flat cells (1e-4, 3e-4) are the candidates to deepen.
  2. Where is the collapse cliff? Between σ=1e-3 and 3e-3 for giant/scope-all on this task. Any future RandOpt variant (structured perturbation, trained-head center) must search below it.
  3. Do the gradient baselines separate? Not at this budget: probe ≈ FT-full ≈ FT-last4 ≈ 77.8%. Adaptation is head-dominated; moving 1.138B backbone weights buys ≤ 0.03pp over training 1.5M head weights. This convergence sets a concrete target: RandOpt "working" here means approaching +2.2pp without gradients.

Open items

① Probe rerun decision — mini-batch vs more full-batch epochs (seconds of compute, ~0.7pp of honesty). ② RandOpt next tier — deepen σ ≤ 3e-4 cells toward paper-N via pooled seed-disjoint jobs, or pivot to a trained-head center; both gated on the train/test split review with Nimrod. ③ FT and RandOpt extension to the other three corruptions. ④ Engine bf16 inference — would roughly halve RandOpt's per-perturbation cost before any large campaign.
