RandOpt / Neural Thickets

Experiment Report — Random Perturbation as an Alternative to Fine-Tuning
Generated July 1, 2026  |  W&B project: noyhassid-tau/randopt

1. What Was Tested

Neural Thickets (arXiv 2603.12228) proposes that high-quality task-adapted models are densely packed around pretrained weights in parameter space. Rather than fine-tuning with gradients, RandOpt randomly perturbs weights using Gaussian noise, scores each perturbation on a small training set, selects the top-K, and uses majority voting (ensemble) for inference.

This session tested the hypothesis on two distinct model families:

🔤 LLM Track

Task: Countdown arithmetic
Models: Qwen2.5-0.5B and 1.5B Instruct
Metric: % of problems solved correctly

🖼 Vision Track

Task: CIFAR-10 image classification
Model: DINOv2-base + linear probe
Metric: top-1 accuracy

Countdown task Given 4 numbers and a target value, produce a valid arithmetic expression (using +, −, ×, ÷) that evaluates to the target. This is a pure reasoning task with no memorizable patterns. A random model scores ~0–13% depending on size.

2. Algorithm Phases

Phase 1 — Base eval: Run the unmodified model on a train split and a held-out test split. Records base/train_reward and base/test_accuracy. This is the baseline to beat.
Phase 2 — Perturbation sampling: For each of population_size random (seed, σ) pairs, add Gaussian noise N(0, σ²) to every weight, run inference on the train set, then subtract the exact same noise (deterministic per-seed generator). Records sampling/batch_mean_reward per batch.
Phase 3 — Selection: Sort all perturbations by train reward. Keep the top-K according to top_k_ratios × population. Records sampling/best_sigma — the σ value of the best-performing perturbation.
Phase 4 — Ensemble evaluation: Re-apply each top-K perturbation on the test set. Majority-vote predicted answers per question across the K models. Reports ensemble/kK/accuracy and ensemble/kK/gain_over_base.

3. Engine Architecture — LLM vs Vision

Design principle Both engines wrap a pretrained model in a Ray remote actor on a single GPU and implement the same three-operation contract: perturb_weights(seed, σ), run inference, restore_weights(seed, σ). The perturbation math is bit-for-bit identical — only the inference path differs.

🔤 LLM Engine — core/engine.py

Built on vLLM + Ray placement groups

Model: Any HuggingFace causal LM (Qwen, LLaMA…). Loaded by vLLM into GPU shards.
Perturb target: All LM weights (0.5B–7B+ params). Accessed via WorkerExtension injected into vLLM's worker class.
Inference: Text generation with sampling params → string outputs → regex answer extraction.
Ensemble: Majority vote across K string answers per question.
Multi-GPU: Tensor parallel (tp > 1) via vLLM; multiple engines = multiple model copies.

🖼 Vision Engine — vision/engine.py

Built on HuggingFace Transformers + Ray actor

Model: DINOv2 backbone (86M params) + optional linear head. Loaded directly via Dinov2Model.from_pretrained().
Perturb target: All backbone + head weights (perturb_target="all") or head only ("classifier").
Inference (classification): forward() → CLS token → linear head → argmax class ID.
Inference (correspondence): get_patch_features() → 16×16 L2-normalized patch tokens → cosine similarity matrix.
Ensemble: Average (N, 256, 256) similarity matrices across K models, then argmax per keypoint.

Perturbation — Shared Math, Different Access Path

LLM — via WorkerExtension injected into vLLM

# utils/worker_extn.py
def perturb_self_weights(self, seed, sigma):
    for name, p in model.named_parameters():
        gen = torch.Generator(device=p.device)
        gen.manual_seed(seed)
        noise = torch.randn(p.shape, generator=gen)
        p.data.add_(sigma * noise)

def restore_self_weights(self, seed, sigma):
    for name, p in model.named_parameters():
        gen = torch.Generator(device=p.device)
        gen.manual_seed(seed)  # identical noise
        noise = torch.randn(p.shape, generator=gen)
        p.data.add_(-sigma * noise)

Vision — direct PyTorch parameter loop

# vision/engine.py
def perturb_weights(self, seed, sigma):
    for _name, p in self._perturb_params():
        gen = torch.Generator(device=p.device)
        gen.manual_seed(seed)
        noise = torch.randn(p.shape, generator=gen)
        p.data.add_(sigma * noise)

def restore_weights(self, seed, sigma):
    for _name, p in self._perturb_params():
        gen = torch.Generator(device=p.device)
        gen.manual_seed(seed)  # identical noise
        noise = torch.randn(p.shape, generator=gen)
        p.data.add_(-sigma * noise)
Why per-param generators? Each parameter gets its own torch.Generator re-seeded with the same seed. This guarantees restore = undo exactly — no floating-point drift, even across multiple perturb/restore cycles. The LLM path uses this to avoid re-loading weights from disk between perturbations. The vision path inherited the same design.

7. SSL Correspondence — DINOv2 + SPair-71k (Thicket Search)

Setup The cleanest SSL test: semantic correspondence on SPair-71k using raw DINOv2 patch embeddings — no linear head, no training of any kind. Reward = PCK@0.1 (a source keypoint's most-similar target patch must land within 10% of the target bounding box). This tests whether random weight perturbation can find a better correspondence model in the neighborhood of the SSL init. Base PCK@0.1 ≈ 58% (headroom exists — supervised SOTA ~80–90%).
Why the earlier ensemble looked flat — and the honest metric A first N=500 run gave ≈0 ensemble gain, and the best single perturbation scored +2pp on the scoring set but +0 on held-out (classic winner's-curse). So we switched to an honest design: each perturbation is scored on two disjoint held-out splits A and B; we rank on A and report the winner's score on B (unbiased "one-expert" gain), and track ρ(A,B) — the selection-generalization signal (ρ≈0 = pure noise, ρ→1 = real reproducible effect).

Verdict: no thicket — hypothesis (b) confirmed DINOv2's SSL init sits on a flat plateau for frozen-backbone / no-head correspondence, not in a thicket of better models. In the safe regime (low σ or concentrated scope) the mean perturbation ≈ base and the tail never rises above noise; where perturbations have real reproducible effect (high ρ — e.g. all σ=1e-3, ρ=0.63) the effect is negative. Quadrupling to N=3000 at the best cells did not uncover an expert. Unlike an instruct-LLM (already task-adapted, surrounded by good nearby solutions), a raw SSL backbone is not near a cluster of better task models that random search can reach.
→

Implication for future work

To make RandOpt work on SSL, give it a trainable component to perturb around — a trained correspondence head or a partial last-block fine-tune — so the base already sits in a task-thicket. Perturbing the raw SSL backbone is a confirmed dead end. The scope→cliff mechanism itself is real (concentrating on the last block avoids collapse), so a hybrid "train a little, then thicket-search the last block" is the natural next experiment.

9. Key Findings

①

Neural Thickets confirmed in LLMs

RandOpt delivers large accuracy gains purely from random perturbation — no gradients, no labelled training beyond reward scoring. 0.5B: 24× gain (0.5%→12%). 1.5B: 3.1× gain (13%→39.85%). The effect is robust and reproducible at N=500.

②

Larger models benefit more in absolute terms

The 1.5B model gains +26.85 pp vs +11.65 pp for 0.5B. Bigger models have richer "thickets" — the density of near-optimal perturbations scales with model capacity. Best sigma is identical (σ=0.0005) across model sizes, suggesting the optimal noise scale is a property of the pretraining regime, not architecture size.

③

K=25 captures most ensemble value; K=50 adds little

For both LLM runs, going from K=25 to K=50 added only 0.2–0.6 pp. The top-25 perturbations already cover the diversity of vote-able answers. Running K=50 costs 2× the compute of K=25 for diminishing returns.

④

Vision: ceiling effect when starting near-optimal

DINOv2-base + trained linear probe reaches 98.7% baseline. Random perturbations find σ=0.0001 regions averaging 99.04% train reward — technically better than base — but the ensemble only gains +0.05 pp on test. The model is already near the information ceiling of CIFAR-10 (32×32 pixels, 10 classes). RandOpt's value proposition requires headroom above the base model.

⑤

Vision weights are far more brittle to noise than LLM weights

DINOv2 shows a cliff edge: σ=0.001 → 95%, σ=0.01 → 10% (random). For LLMs, σ=0.001 → graceful 30–40% relative drop. DINOv2 backbone features are geometrically precise — small perturbations preserve them, moderate ones destroy them entirely. This has practical implications: vision RandOpt requires σ ≤ 10⁻⁴ for meaningful exploration.

⑥

Warm init is critical for vision; random head invalidates the experiment

The smoke test (random head, 8% base) shows σ=0.1 as best — but this is misleading. The "perturbations" are effectively training the head from scratch, not exploring the pretrained model's neighborhood. A proper vision Neural Thickets experiment requires a starting point with meaningful task performance. The linear probe (98.22%) provides this, making the experiment analogous to the LLM setting where the base model already has task capability.

10. Recommended Next Steps

LLM Track

  • Scale to 3B and 7B models — test if the absolute gain keeps growing
  • Explore σ < 0.0005 (e.g. 0.0001, 0.0002) — best sigma hit the lower floor in both runs
  • Scale population to N=5000 to see if the gain curve saturates
  • Try GSM8k as a second reasoning task — cross-task generalization of best sigma

Vision Track

  • Try a harder task where the base model is not near-ceiling (e.g. DINOv2 on fine-grained datasets: CUB-200, FGVC-Aircraft where linear probe ~80%)
  • Restrict perturbations to linear head only (backbone frozen) to test if thicket effect lives in the head or backbone
  • Try σ range 10⁻⁵ to 10⁻⁴ — current best sigma 0.0001 is already at the explored floor
  • Scale to DINOv2-large to check if the cliff edge shifts
